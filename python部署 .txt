================================================================================
Windows 服务器部署后端 — 完整步骤
================================================================================

前置假设
--------
- Windows 服务器（管理员权限）
- 有公网 IP，且能 SSH/远程桌面登录
- 已有可用的 LLM API Key（你本地 config.json 里那个）


================================================================================
步骤 1：装 Python 3.12
================================================================================

1. 浏览器打开 https://www.python.org/downloads/release/python-31210/
2. 下拉到底部 Files，下载 Windows installer (64-bit) — python-3.12.10-amd64.exe
3. 双击安装，勾选两个关键选项：
   ✅ Add python.exe to PATH
   ✅ Install for all users（如有）
4. 装好后开 PowerShell 验证：

   python --version
   # 应输出 Python 3.12.x


================================================================================
步骤 2：装 PostgreSQL 14+
================================================================================

1. 打开 https://www.postgresql.org/download/windows/
2. 下载 EnterpriseDB 安装包，最新稳定版（16.x 都行）
3. 安装时记下：
   - 安装路径（默认 C:\Program Files\PostgreSQL\16）
   - postgres 用户密码（重要，后面 config.json 要用）
   - 端口（默认 5432）
4. 安装完毕后，开始菜单找到 pgAdmin 4，登录
5. 在 pgAdmin 里新建数据库：
   - 右键 Databases → Create → Database
   - Database name: doc_review
   - Owner: postgres
   - 保存


================================================================================
步骤 3：在你本地（开发机）打包发布包
================================================================================

1. 用 PowerShell 进入项目根目录：

   cd e:\qoderproject\20260527

2. 运行打包脚本：

   powershell -ExecutionPolicy Bypass -File deploy\pack-backend.ps1

3. 脚本输出在 release\doc-audit-YYMMDD-HHMM.zip
   大小约 100 KB，含 34 个文件（仅源码 + 迁移 + 配置模板，无虚拟环境、
   无敏感数据）

4. 把这个 zip 通过下面任意方式上传到服务器：
   - 远程桌面 → 直接拖拽到服务器桌面
   - 网盘共享（百度云、腾讯微云、企业网盘）
   - SCP/SFTP（如服务器开了 SSH）
   - 邮件/微信发自己（小文件可行，100 KB 没问题）


================================================================================
步骤 4：在服务器上解压
================================================================================

1. 在服务器上创建目录（PowerShell）：

   mkdir C:\apps

2. 把 zip 文件复制到 C:\apps\，假设文件名为 doc-audit-260617-1756.zip

3. 解压：

   cd C:\apps
   Expand-Archive doc-audit-260617-1756.zip -DestinationPath doc-audit
   cd doc-audit

4. 验证目录结构：

   dir
   # 应看到 backend / migrations / config.json.example / requirements.txt
   # / alembic.ini / python部署 .txt 等


================================================================================
步骤 5：创建 Python 虚拟环境 + 装依赖
================================================================================

   cd C:\apps\doc-audit

   # 创建虚拟环境
   python -m venv .venv312

   # 激活
   .venv312\Scripts\activate
   # 终端提示符前面会出现 (.venv312) 字样

   # 装依赖（用清华镜像，国内快得多）
   pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

这一步慢：PaddleOCR + paddlepaddle 加起来约 700 MB，国内带宽 5-15 分钟。装完后
磁盘多约 1.1 GB。

如果遇到 Could not find a version that satisfies the requirement 错误，多半是某
个包不支持 Python 3.12，把那个包跳过：

   pip install -r requirements.txt --no-deps -i https://pypi.tuna.tsinghua.edu.cn/simple


================================================================================
步骤 6：创建配置文件
================================================================================

   copy config.json.example config.json
   notepad config.json

按你的实际情况填写，重点 4 项：

   {
     "database": {
       "host": "localhost",
       "port": 5432,
       "user": "postgres",
       "password": "你刚才装 PostgreSQL 时设置的 postgres 密码",
       "dbname": "doc_review"
     },
     "llm": {
       "api_key": "你本地 config.json 里那个 ark-... key",
       "base_url": "https://ark.cn-beijing.volces.com/api/v3",
       "model": "你本地 config.json 里那个 ep-... model id",
       "temperature": 0.1
     },
     "max_ocr_pages": 5,
     "document_types": [
       "身份证", "护照", "户口本", "结婚证", "出生证明",
       "学历证书", "学位证书", "工作证明", "银行流水",
       "存款证明", "房产证", "无犯罪证明"
     ]
   }

保存关闭。


================================================================================
步骤 7：初始化数据库表
================================================================================

虚拟环境仍激活的状态：

   cd C:\apps\doc-audit
   alembic upgrade head

应该看到一连串 INFO [alembic.runtime.migration] Running upgrade ... 输出，最后
到 007_summary_relevance。

验证表已建好：在 pgAdmin 里展开 doc_review → Schemas → public → Tables，应能看
到 8 张表（clients / documents / family_members / assets / templates /
template_fills / split_tasks / summaries / client_info / alembic_version）。


================================================================================
步骤 8：开放 Windows 防火墙端口
================================================================================

后端将监听 8000 端口，公网要访问必须开防火墙。用管理员权限开 PowerShell：

   New-NetFirewallRule -DisplayName "DocAudit Backend 8000" `
     -Direction Inbound -Protocol TCP -LocalPort 8000 -Action Allow

返回一条规则信息表示成功。

如果你的服务器在云厂商（阿里云/腾讯云/AWS），还要在云控制台的安全组里也开放
8000 端口入站，否则 Windows 防火墙开了也没用。


================================================================================
步骤 9：启动后端
================================================================================

虚拟环境仍激活：

   cd C:\apps\doc-audit\backend

   # 关键：必须从 backend 目录启动，且加这两个环境变量防止 GBK 编码报错
   $env:PYTHONIOENCODING="utf-8"
   $env:PYTHONUTF8="1"
   ..\.venv312\Scripts\python.exe -m uvicorn main:app --host 0.0.0.0 --port 8000

应该看到：

   INFO:     Started server process [...]
   INFO:     Waiting for application startup.
   数据库连接池已初始化
   配置已加载，数据库已连接，服务启动完成
   INFO:     Application startup complete.
   INFO:     Uvicorn running on http://0.0.0.0:8000

首次启动慢：PaddleOCR 第一次会从 ModelScope 下载 PP-OCRv4 模型（~150 MB），大约
1-3 分钟。等"服务启动完成"出现就 OK。

关键提醒：这个 PowerShell 窗口关掉服务就停了。后端服务化下次再说，先确保它能
跑起来。


================================================================================
步骤 10：测试
================================================================================

10.1 服务器本地验证
-------------------
新开一个 PowerShell 窗口（不要关原来那个跑 uvicorn 的）：

   curl http://localhost:8000/api/clients
   # 应返回 {"clients":[],"total":0}

   curl http://localhost:8000/docs
   # 应返回 200，是 Swagger 文档页面 HTML

10.2 公网验证
-------------
在你的本机笔记本打开浏览器：

   http://服务器公网IP:8000/docs

应能看到 Swagger 文档，43 个 API 接口列表（含中文 docstring）。

测试调用文件解析接口（先要有公网可访问的 URL）：

PowerShell 本机：
   Invoke-WebRequest -Uri "http://服务器公网IP:8000/api/file-summary" -Method POST `
     -ContentType "application/json" `
     -Body '{"url":"https://www.adobe.com/jp/print/pdfs/PDF-Tour.pdf","progress_name":"测试进展"}'

Mac/Linux 本机：
   curl -X POST http://服务器公网IP:8000/api/file-summary \
     -H "Content-Type: application/json" \
     -d '{"url":"https://www.adobe.com/jp/print/pdfs/PDF-Tour.pdf","progress_name":"测试进展"}'

返回的 JSON 应含 summary / key_points / relevance 字段。耗时 30-200 秒（OCR + LLM）。


================================================================================
后端跑起来后能直接看到的
================================================================================

地址                                  内容
------------------------------------  --------------------------------------------
http://IP:8000/docs                   Swagger 接口文档（最快验证后端活着）
http://IP:8000/redoc                  ReDoc 接口文档（另一种风格）
http://IP:8000/openapi.json           OpenAPI 规范 JSON
http://IP:8000/api/...                实际业务接口

注意：你没部署前端，所以 http://IP:8000/ 根路径会返回 404。这是正常的——前端等
你说"也部署"再给前端的步骤。


================================================================================
关闭后端
================================================================================

回到跑 uvicorn 的 PowerShell 窗口，按 Ctrl+C 即可。


================================================================================
出问题怎么办
================================================================================

症状                              排查
--------------------------------  ----------------------------------------------
python 命令找不到                  Python 安装时没勾 PATH，重新安装
alembic upgrade head 报连接失败    config.json 里 PostgreSQL 密码/host/port 不对
                                   或 PostgreSQL 服务没启动
uvicorn 启动后 print 报            漏了 $env:PYTHONIOENCODING="utf-8" 这两行
UnicodeEncodeError
公网 IP 访问 8000 不通             1) Windows 防火墙规则
                                   2) 云厂商安全组
                                   3) 服务器是否真有公网 IP
OSError: [WinError 10013] 端口     别的程序占了 8000，换 --port 8001 或杀进程
被占用
PaddleOCR 卡很久                   首次下载模型，等 3 分钟；如果一直卡，看
                                   PowerShell 输出有没有"模型下载失败"之类提示


================================================================================
后续更新流程（重要！）
================================================================================

本地开发改了代码后想更新到服务器：

1. 在本地重新跑 deploy\pack-backend.ps1，生成新 zip
2. 把新 zip 上传到服务器
3. 在服务器：
   - 备份当前版本（可选）：Rename-Item C:\apps\doc-audit C:\apps\doc-audit-bak
   - 创建新目录：mkdir C:\apps\doc-audit-new
   - 解压新 zip 到 doc-audit-new
   - 把 config.json 从老目录拷过来（不要重新填）：
     copy C:\apps\doc-audit-bak\config.json C:\apps\doc-audit-new\
   - 把虚拟环境也拷过来（节省重装时间）：
     xcopy /E /I C:\apps\doc-audit-bak\.venv312 C:\apps\doc-audit-new\.venv312
   - 切换：
     stop-service ... 或 Ctrl+C 停掉旧后端
     Rename-Item C:\apps\doc-audit C:\apps\doc-audit-old
     Rename-Item C:\apps\doc-audit-new C:\apps\doc-audit
   - 跑数据库迁移（如有新表/新列）：
     cd C:\apps\doc-audit
     .venv312\Scripts\activate
     alembic upgrade head
   - 重启后端

更省心的做法：未来要常更新可以装 git，然后 git pull 就够了。
